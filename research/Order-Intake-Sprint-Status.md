## One-line summary

**Read + judgment + persist + confirm + orchestrate are complete.** The entire `.eml → EmailEnvelope → (ClassifiedDocument) → ParsedDocument → ValidationResult → (ClarifyEmail) → ProcessResult → (ConfirmationEmail) → RunSummary` pipeline runs end-to-end through the assembled `SequentialAgent('order_intake_pipeline')` at `backend/my_agent/agent.py`. `adk web adk_apps` discovers it via the `adk_apps/order_intake/` single-entry scan dir (post-F1); `adk eval adk_apps/order_intake tests/eval/smoke.evalset.json --config_file_path tests/eval/eval_config.json` exercises 3 live cases. The Step 6 integration test drives the AUTO_APPROVE path through `Runner.run_async` against the Firestore emulator deterministically (stubbed LlmAgents, including a ConfirmStage child-LlmAgent stub post-f5db946).

**What's left is the surface half** — Track A2 (Gmail send), Track A3 (push-based ingestion), Track B (Generator-Judge quality gate), Track E (Embedding Tier 3 + expanded evalsets), Track Demo (`scripts/run_demo.py`), read-only dashboard, and the Post-MVP items roadmapped in `Glacis-Order-Intake.md` (Learning Loop, auto-merge reply, cloud deploy). **Track A1 (Gmail polling ingress) landed 2026-04-25** — `backend/gmail/` package + `scripts/gmail_auth_init.py` + `scripts/gmail_poll.py`; 19 new unit tests + 1 gated live integration; pipeline now reachable from a real Gmail inbox via 30s poll loop. No in-sprint blockers remain.
